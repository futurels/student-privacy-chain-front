# 基于区块链的高校学生隐私数据存证与共享系统
## 数据库设计文档

**版本号：** V1.0  \
**数据库建议：** MySQL 8.0  \
**设计依据：** 任务书、开题报告、需求分析文档、接口文档

## 1. 设计背景与目标

本数据库围绕“数据存证、授权共享、追溯审计、隐私保护”四大核心模块设计，采用“链上存摘要、链下存密文、IPFS 存文件、MySQL 存索引与业务状态”的协同思路。数据库既要支撑 40+ API 的业务落地，也要满足四级权限控制、隐私数据不上链、授权可撤销、全流程可追溯等要求。

## 2. 总体设计说明

- 字符集：utf8mb4
- 排序规则：utf8mb4_0900_ai_ci
- 存储引擎：InnoDB
- 主键策略：BIGINT UNSIGNED + AUTO_INCREMENT
- 删除策略：核心主数据采用逻辑删除；日志与链上索引表保留物理历史
- 链上链下划分：链上保存摘要、授权、审计与交易回执；链下保存原始密文、结构化业务内容与文件实体

## 4. 表间关系概述

- sys_department 1:N sys_user：一个部门可对应多个用户。
- sys_user M:N sys_role：通过 sys_user_role 建立用户角色关系。
- sys_role M:N sys_permission：通过 sys_role_permission 建立角色权限关系。
- sys_user 1:1 student_profile：学生账号与学生基本信息一一对应。
- student_profile 1:N student_status / student_grade / privacy_data：一个学生可拥有多条学籍、成绩和隐私数据记录。
- privacy_data 1:N privacy_file：一条隐私数据可挂接多个文件。
- privacy_data 1:N evidence_application / authorization_application：一条隐私数据可发起多次存证或授权申请。
- evidence_application 1:1 evidence_record：一个通过审核的存证申请最终形成一条存证记录。
- authorization_application 1:1 authorization_record：一个通过审核的授权申请最终形成一条授权记录。
- authorization_record 1:N access_log：一条授权可对应多次访问行为。
- blockchain_tx_record 1:1（逻辑上） evidence_record 或 authorization_record：链上回执由业务表引用。
- privacy_data / evidence_record / authorization_record 可选关联 zk_proof_record：用于记录增强型隐私校验。

## 6. 完整建表语句

```sql
-- ============================================================
-- 基于区块链的高校学生隐私数据存证与共享系统
-- MySQL 8.0 初始化脚本
-- 说明：采用 utf8mb4 字符集，存储引擎为 InnoDB。
-- ============================================================

CREATE DATABASE IF NOT EXISTS `student_privacy_chain`
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_0900_ai_ci;

USE `student_privacy_chain`;

CREATE TABLE `sys_department` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '部门主键',
  `parent_id` BIGINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '父级部门ID，根节点为0',
  `dept_code` VARCHAR(64) NOT NULL COMMENT '部门编码，系统内唯一',
  `dept_name` VARCHAR(100) NOT NULL COMMENT '部门名称',
  `dept_type` VARCHAR(32) NOT NULL DEFAULT 'ORG' COMMENT '部门类型，如 COLLEGE、ACADEMIC、IT_CENTER、ORG',
  `leader_name` VARCHAR(50) NULL COMMENT '负责人姓名',
  `contact_phone` VARCHAR(20) NULL COMMENT '联系电话',
  `sort_no` INT NOT NULL DEFAULT 0 COMMENT '显示排序号',
  `status` VARCHAR(16) NOT NULL DEFAULT 'ENABLED' COMMENT '状态，如 ENABLED、DISABLED',
  `remark` VARCHAR(255) NULL COMMENT '备注说明',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间，自动维护',
  `deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '逻辑删除标记，0未删，1已删',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_sys_department_code` (`dept_code`),
  KEY `idx_sys_department_parent` (`parent_id`),
  KEY `idx_sys_department_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='部门表';

CREATE TABLE `sys_user` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '用户主键',
  `username` VARCHAR(64) NOT NULL COMMENT '登录账号；学生可用学号，教师/管理员可用工号或系统账号',
  `password_hash` VARCHAR(255) NOT NULL COMMENT '密码摘要，不保存明文',
  `password_salt` VARCHAR(64) NULL COMMENT '密码盐值',
  `user_type` VARCHAR(32) NOT NULL DEFAULT 'STUDENT' COMMENT '用户类型，如 STUDENT、STAFF、ADMIN、SYSTEM',
  `real_name` VARCHAR(50) NOT NULL COMMENT '真实姓名',
  `gender` VARCHAR(16) NULL COMMENT '性别',
  `phone` VARCHAR(20) NULL COMMENT '手机号',
  `email` VARCHAR(100) NULL COMMENT '邮箱',
  `dept_id` BIGINT UNSIGNED NULL COMMENT '所属部门ID，关联 sys_department.id',
  `status` VARCHAR(16) NOT NULL DEFAULT 'ENABLED' COMMENT '账户状态，如 ENABLED、DISABLED、LOCKED',
  `last_login_at` DATETIME NULL COMMENT '最近登录时间',
  `last_login_ip` VARCHAR(64) NULL COMMENT '最近登录IP',
  `remark` VARCHAR(255) NULL COMMENT '备注说明',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间，自动维护',
  `deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '逻辑删除标记',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_sys_user_username` (`username`),
  KEY `idx_sys_user_dept` (`dept_id`),
  KEY `idx_sys_user_status` (`status`),
  CONSTRAINT `fk_sys_user_dept` FOREIGN KEY (`dept_id`) REFERENCES `sys_department` (`id`) ON DELETE SET NULL ON UPDATE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='用户表';

CREATE TABLE `sys_role` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '角色主键',
  `role_code` VARCHAR(64) NOT NULL COMMENT '角色编码，如 STUDENT、COUNSELOR、TEACHING_ADMIN、SYS_ADMIN',
  `role_name` VARCHAR(100) NOT NULL COMMENT '角色名称',
  `role_scope` VARCHAR(32) NOT NULL DEFAULT 'SELF' COMMENT '数据范围，如 SELF、DEPT、ALL',
  `status` VARCHAR(16) NOT NULL DEFAULT 'ENABLED' COMMENT '状态',
  `remark` VARCHAR(255) NULL COMMENT '备注说明',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间，自动维护',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_sys_role_code` (`role_code`),
  KEY `idx_sys_role_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='角色表';

CREATE TABLE `sys_permission` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '权限主键',
  `permission_code` VARCHAR(100) NOT NULL COMMENT '权限编码',
  `permission_name` VARCHAR(100) NOT NULL COMMENT '权限名称',
  `permission_type` VARCHAR(32) NOT NULL DEFAULT 'API' COMMENT '权限类型，如 MENU、BUTTON、API、DATA',
  `parent_id` BIGINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '父级权限ID',
  `path` VARCHAR(255) NULL COMMENT '前端路由或接口路径',
  `http_method` VARCHAR(16) NULL COMMENT 'HTTP方法，如 GET、POST',
```
